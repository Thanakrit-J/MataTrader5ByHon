🥇 MT5 Gold AI Trading — สถาปัตยกรรม (แนวทาง B)

หัวใจ: Strategy + Feature + Cost model ชุดเดียว ใช้ร่วมกันทั้ง Backtest และ Live → "สิ่งที่เทสต์ = สิ่งที่เทรดจริง" (ไม่มี train/live skew)

1 · DATA
MT5 Terminalดึงประวัติ XAUUSD ย้อนหลังหลายปี (M1/M5/M15/H1) ผ่าน copy_rates_range
Data Pipelineทำความสะอาด · จัดการ gap/วันหยุด · resample timeframe · ตรวจสอบความถูกต้อง
Data Store (Parquet)ไฟล์ columnar เร็ว โหลดซ้ำได้ + คง PostgreSQL เดิมไว้ log เทรดสด
▼
2 · SHARED CORE (ใช้ร่วมกัน)
◆ โค้ดชุดเดียวที่ทั้งฝั่งวิจัย (ML) และฝั่งเทรดจริงเรียกใช้
Feature EngineeringEMA/RSI/ATR · โครงสร้างราคา · zone · session/เวลา · volatility regime
Strategy Interfaceรับ features → คืนสัญญาณ BUY/SELL/FLAT · เสียบได้ทั้ง rule-based และ ML
Cost Modelspread + commission + slippage จริง — จุดที่บอทเดิมมองข้าม
▼
Backtest Engine (event-driven) เดินทีละแท่งบนแท่งที่ ปิดแล้ว · จำลอง SL/TP/trailing · หักต้นทุนทุกไม้ → รายงาน metric: Win rate · Profit Factor · Expectancy · Max Drawdown · Sharpe · จำนวนไม้
3 · ML PIPELINE (research)
Triple-Barrier Labelingติดป้ายแต่ละแท่ง: ราคาชน TP ก่อน / ชน SL ก่อน / หมดเวลา → เป้าหมายที่ตรงกับการเทรดจริง
LightGBM / XGBoostโมเดล tabular บน CPU — เร็ว แม่น ตีความได้ (feature importance)
Walk-Forward Validationกัน overfitting/lookahead · train อดีต → test อนาคตที่ยังไม่เห็น เลื่อนหน้าต่างไปเรื่อยๆ
Model Artifactค่าความน่าจะเป็น P(TP ก่อน SL) + เกณฑ์ความมั่นใจขั้นต่ำ
4 · LIVE EXECUTION (demo)
Live Bot (ซ่อมแล้ว)แก้ NameError, ดึงแท่งล่าสุด, position sizing, ตัด "เดาแล้วเข้าทุกแท่ง"
เรียก Shared Core + Modelใช้ Feature/Strategy/Cost ชุดเดียวกับ backtest → ผลตรงกัน
MT5 Order (demo เท่านั้น)ล็อคความปลอดภัย · ส่งออเดอร์ · จัดการ SL/TP/trailing
Trade Log → Data Storeบันทึกผลจริงกลับเข้าคลังข้อมูล เพื่อวิเคราะห์และเทรนรอบถัดไป
🔁 Feedback loop: ผลเทรดสด (demo) ไหลกลับเข้า Data Store → เทียบกับที่ backtest ทำนายไว้ → ปรับ feature/label/threshold → เทรนใหม่ → พิสูจน์ว่า "พลาดน้อยลง" ด้วยตัวเลข ก่อนตัดสินใจเรื่องเงินจริง
Data Shared Core ML / Research Live Execution โค้ดที่ backtest & live ใช้ร่วมกัน
